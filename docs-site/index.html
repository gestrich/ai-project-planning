Project Planning

A plugin for Claude Code and OpenAI Codex that helps you manage a project — capture brain dumps, see what's in motion, plan the week, and review PRs against documented conventions.

How it works

Each project gets its own local repo. The repo represents your project: it's where your notes and local plans live, and it's the gateway the skills operate through. If you don't have one yet, the bootstrap skill creates and organizes it for you.

From there, you chat about your project the way you would with a teammate — paste a brain dump, ask what's next, plan the week, hand over a PR for review. The skills know about the external sources of truth your project already uses (Confluence pages, Jira issues, Slack channels, GitHub repos) and read from them on demand, so the local repo doesn't duplicate what already lives there. It holds what's not in those systems: in-incubation ideas, personal items, raw transcripts.

Source discovery is driven by the project's AGENTS.md, written in whatever shape makes sense. See Configuration for the details.

Install

The plugin installs on both Claude Code and Codex. Follow the instructions in the repo README:

  • Install on Claude Code
  • Install on OpenAI Codex

Once installed, run the bootstrap skill in an existing project to wire it up. See the Getting Started walkthrough for an end-to-end example.

Skills

Each skill is triggered by natural-language prompts — you don't need to name the skill. Pick one to read more.

bootstrap

Survey an existing project, propose a plan-of-changes, and onboard it to the planning convention.

plans

One skill, four modes: capture brain dumps into notes/, show what's in motion, plan the week, and reconcile local plans against Jira/Confluence.

docs-update

Sweep last week's discussion for things that should land in the docs.

pr-review

Review a GitHub PR against the project's documented conventions and surface doc gaps the PR exposes.
