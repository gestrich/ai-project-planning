Project Planning

A plugin for Claude Code and OpenAI Codex that gives you a consistent gateway into any project — capture notes, navigate plans, plan sprints, sync with external sources of truth, and review PRs against your own documentation.

What it is

Project Planning is a project-agnostic plugin. Drop it into a new project, declare a short ## Project Planning section in the project's AGENTS.md, and the skills figure out which sources apply (Jira, Confluence, a local plans/ folder, Slack-as-context, or some combination) and read only what's relevant.

Every skill is read-by-default. Any external write — a Jira transition, a Confluence edit, a Slack send, a PR comment, a local file change — only happens after you explicitly confirm it. The skills surface decisions; they don't make them.

Install

The plugin installs on both Claude Code and Codex. Follow the instructions in the repo README:

  • Install on Claude Code
  • Install on OpenAI Codex

Once installed, run the bootstrap skill in an existing project to wire it up. See the Getting Started walkthrough for an end-to-end example.

Skills

Each skill is triggered by natural-language prompts — you don't need to name the skill. Pick one to read more.

notes

Capture raw voice transcripts and brain dumps verbatim into the project's notes/ folder.

plan

Synthesize what's in motion across the project's planning sources — and reconcile local plans against external sources of truth.

sprint

Coach you through picking what to do this week, then draft a sprint file once you sign off.

bootstrap

Survey an existing project, propose a plan-of-changes, and onboard it to the planning convention.

docs-update

Sweep last week's discussion for things that should land in the docs.

pr-review

Review a GitHub PR against the project's documented conventions and surface doc gaps the PR exposes.
