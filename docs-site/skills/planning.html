Skill

planning

One skill, four modes. The skill picks the mode from your ask — capture, status, sprint, or reconciliation — and reads your AGENTS.md to discover which planning sources apply.

Reads
The project's AGENTS.md, notes/, plans/ (manifest plus domain documents), plans/sprints/, and whichever external sources AGENTS.md declares — Jira, Confluence, Slack as read-only context.
Writes
Capture mode writes a single file to notes/<YYYY-MM-DD>-<slug>.md. Sprint mode writes plans/sprints/<YYYY-MM-DD>.md after you approve the draft. Status and reconciliation modes don't write anything on their own; any external mutation routes through the source-specific reference document and requires explicit confirmation.

The four modes

Capture mode

Triggers when you paste a long, unstructured, first-person block that reads like a spoken transcript — or when you explicitly say "here's my brain dump", "voice transcript", "dump this into notes", "capture this".

Writes the transcript verbatim to notes/<YYYY-MM-DD>-<slug>.md. No cleanup, no rewriting, no summarization. The raw form is the artifact.

Status mode

Triggers on "what's the plan", "what's next", "catch me up", "where are we on X", "what's the status". Synthesizes what's in motion across the configured planning sources.

Output is source-cited (Jira issue keys, Confluence page links, local file paths, Slack permalinks), grouped by area not by source, with asks aimed at you surfaced at the top. Short, oriented for decision-making — not a full report.

Sprint mode

Triggers on "what should I do next", "plan my sprint", "what's on deck this week", "what should I focus on". Acts as a scrum-master coach.

Gathers context from recent notes, the configured planning sources, the previous sprint file, and the configured Slack channel. Asks challenging questions about what's load-bearing before drafting plans/sprints/<YYYY-MM-DD>.md. Surfaces decisions; doesn't make them. Writes only after you approve the draft.

Reconciliation mode

Triggers on "sync plans", "what's drifted", "what should graduate", "is the local plan still aligned with Jira/Confluence". A hygiene pass.

Compares the local plans/ folder against the configured external sources of truth and reports three things, side-by-side: graduation candidates (firm enough to live externally now), drift (local and external disagree), and stale local items (overtaken by external decisions). Surfaces drift; never silently resolves it.

How it reads AGENTS.md

The skill looks for planning-relevant information in AGENTS.md in whatever shape you've written it — no required heading, no enforced schema. Signals it picks up:

  • A Jira project key, epic ID, or board URL → loads jira.md.
  • A Confluence space key, page ID, or page URL → loads confluence.md.
  • A Slack channel name or ID flagged as project context → loads slack.md.
  • A mention of plans/ as a source, or no external system named → loads local.md.
  • A list of plugin skills the project uses, with one-line "what it's for / when to use it" notes — honored when deciding what's in scope.

If AGENTS.md doesn't exist or has no planning-relevant content, the skill says so and offers to run bootstrap — it never guesses the planning shape from the repo contents.

Reference documents

The skill loads only the reference documents that match the configured sources, keeping context lean. Each lives in plugin/skills/planning/:

  • notes.md — capture mode mechanics.
  • sprint.md — sprint-coaching workflow and sprint file shape.
  • jira.md — Jira read/write discipline plus a reconciliation subsection.
  • confluence.md — Confluence read-before-write workflow plus a reconciliation subsection.
  • slack.md — Slack as read-only context capture.
  • local.md — the plans/ folder convention (manifest + domain documents + sprints) plus a reconciliation subsection.
