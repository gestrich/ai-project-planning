Skill

plans

The main entry point for managing your project. Capture brain dumps, ask what's in motion, plan the week, and reconcile your local plans against Jira/Confluence — all from one skill.

What it does

Capture a brain dump

Paste a transcript or first-person rambling — voice notes, Apple Notes, anything in that shape — and it lands as a dated markdown file in notes/ verbatim. No cleanup, no rewriting, no summarization. The raw form is the artifact, so later you can ask the skill to extract action items, decisions, or sprint candidates from it.

Try: "here's my brain dump", "voice transcript", or just paste the text.

Ask what's in motion

Synthesizes the current state of your project across whichever sources you use — Jira issues, Confluence pages, the local plans/ folder, recent Slack discussion. The answer is source-cited (Jira issue keys, Confluence page links, file paths, Slack permalinks), grouped by area not by source, with asks aimed at you up top. Short and oriented for decision-making — not a status report.

Try: "what's the plan?", "catch me up", "where are we on X?", "what's next?"

Plan the week

Acts as a scrum-master coach. Gathers context from recent notes, your planning sources, the previous sprint file, and the configured Slack channel — then asks challenging questions about what's load-bearing before drafting a sprint file. The draft is shown inline; the file isn't written until you sign off.

Try: "what should I do next?", "plan my sprint", "what's on deck this week?"

Reconcile drift

A hygiene pass. Compares your local plans/ folder against external sources of truth (Jira, Confluence) and reports three things: graduation candidates (firm enough that they should live externally now), drift (local and external disagree about the same item), and stale local items (overtaken by external decisions). Surfaces drift; never silently resolves it.

Try: "sync plans", "what's drifted?", "what should graduate?", "is the local plan still aligned with Jira?"

What it reads from AGENTS.md

The skill discovers your project's planning sources by reading your AGENTS.md. Free-form markdown — no required heading, no schema. It picks up:

  • Jira — a project key, epic ID, or board URL tells the skill where shipped work lives.
  • Confluence — a space, page ID, or page URL tells the skill where long-form plans and decisions live.
  • Slack — a channel name or ID flagged for context lets the skill pull recent discussion (read-only).
  • Local plans/ folder — declared as a source for in-incubation ideas and personal items.
  • GitHub repos — useful when citing work-in-flight against specific repos.
  • Which plugin skills the project uses, with one-line notes on what each is for in this project. Honored when deciding what's in scope.

If AGENTS.md doesn't exist or has no planning content yet, the skill says so and offers to run bootstrap. It never guesses the planning shape from your repo contents.

See Configuration for an example shape.

Project folder structure

This skill (and bootstrap, which sets it up the first time) uses a small folder layout inside your project:

your-project/
├── AGENTS.md            # planning content lives somewhere in here
├── notes/               # raw transcripts dropped by capture
│   └── YYYY-MM-DD-<slug>.md
└── plans/               # local planning artifacts
    ├── plan.md          # manifest pointing at the other plan files
    ├── <domain>.md       # one file per area of the project (UI, infra, etc.)
    └── sprints/
        └── YYYY-MM-DD.md  # one file per sprint, dated by start of week
  • notes/ — raw inputs. Created on first capture; never edited after writing.
  • plans/ — your local planning artifacts. Either the sole source of truth (if you're not using Jira/Confluence) or supplementary to them. Holds in-incubation ideas and personal items that don't belong in a team-visible system.
  • plans/plan.md — a short manifest that lists the other files in plans/ so the skill can navigate without prescribing a structure.
  • sprints/ — one file per sprint, dated by start-of-week.

Run bootstrap to create these folders in an existing project.

What it writes (always with confirmation)

  • A transcript file when you give it a brain dump (notes/YYYY-MM-DD-<slug>.md).
  • A sprint file when you've approved the sprint draft (sprints/YYYY-MM-DD.md).
  • External edits (a Jira transition, a Confluence page update, etc.) only when reconciliation surfaces a graduation candidate and you explicitly approve the specific change. Treat each acted-on item as its own confirmation.
