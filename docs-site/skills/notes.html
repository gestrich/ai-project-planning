Skill

notes

The simplest entry point in the plugin: capture a raw transcript or brain dump and write it, verbatim, into the project's notes/ folder.

Reads
Nothing. The input is the prompt.
Writes
notes/YYYY-MM-DD-<slug>.md in the project root.
Configuration
None. Works in any project the moment it's installed.

What it does

Notes captured here are raw artifacts, not plans. The skill writes the transcript verbatim — no cleanup, no rewriting, no summarization, no headers added. Downstream skills (plan, sprint) may read these notes later, but notes never edits a note after creating it.

When to reach for it

Explicit triggers:

  • "here's my brain dump"
  • "voice transcript"
  • "dump this into notes"
  • "capture this", "save this transcript"

Implicit triggers: a long, unstructured, first-person block that looks like a spoken transcript — rambling sentences, conversational filler ("um", "so", "yeah"), repeated ideas, no headings. If it reads like you talking to yourself, the skill treats it as a transcript even if you don't name it.

Walkthrough

You paste a brain dump:

here's a quick brain dump on the auth migration — so we've got two options, either flip the
whole org over in one shot or stage it by team, and I keep going back and forth because the
staged version means we have to maintain two code paths for a couple of weeks...

The skill:

  1. Runs date +%Y-%m-%d to get today's date.
  2. Picks a short slug (2–4 words, kebab-case) from the content — e.g., auth-migration-options.
  3. Writes the transcript verbatim to notes/2026-05-24-auth-migration-options.md.
  4. Confirms the path back to you in one sentence.

If notes/ doesn't exist, the skill creates it. If a file with the exact same name already exists, it appends a short suffix (-2) rather than overwriting.

What this skill does not do

  • It does not edit, summarize, restructure, or "clean up" the transcript.
  • It does not extract action items, decisions, or todos. Those belong to downstream skills.
  • It does not modify or delete previously written notes.
  • It does not enforce a filename convention beyond the leading YYYY-MM-DD- date prefix.
