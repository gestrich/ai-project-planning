Getting started

A walkthrough for bootstrapping a fresh project onto the plugin and running the skills end-to-end.

1. Install the plugin

Install on whichever runtime you use — instructions live in the repo README:

  • Install on Claude Code
  • Install on OpenAI Codex

2. Bootstrap an existing project

Open a Claude/Codex session inside the project you want to onboard:

/bootstrap set this project up for the planning plugin

The bootstrap skill runs in three phases:

  1. Survey. Reads AGENTS.md, scans for existing notes/plans folders, peeks at commit history for hints of Jira/Confluence/Slack usage, and lists the project's GitHub remote. Read-only.
  2. Propose. Shows you a plan-of-changes — folders to create, files to move (via git mv), and the planning content it wants to add to AGENTS.md (under whatever heading fits the file you already have). You answer a few clarifying questions inline.
  3. Apply. Only after you say "looks good" / "go ahead" / "apply", the skill makes the changes and summarizes what landed.

After bootstrap finishes, your project has a notes/ folder, a plans/ folder, a sprints/ folder, and planning content in AGENTS.md. See the Configuration page for what that looks like.

3. Capture a brain dump

Paste a transcript or brain dump:

/plans here's my brain dump
yeah so I was thinking about the auth migration, we need to figure out
whether to do the rollout in one shot or stage it, and there's that Jira
ticket about the session token storage that I haven't read yet...

The plans skill's capture mode writes it verbatim to notes/YYYY-MM-DD-<slug>.md and confirms the path. No cleanup, no rewriting.

4. Ask what the plan is

/plans what's the plan?

The plans skill's status mode reads your AGENTS.md, loads only the reference documents matching the sources you declared (Jira, Confluence, local plans/, Slack-as-context), and synthesizes a short, source-cited view of the project's direction.

5. Plan the week

/plans what should I focus on this week?

The plans skill's sprint mode gathers recent notes, reads the plan sources, looks at the previous sprint file (if any), and pulls a week of Slack activity from the configured channel. It then acts as a scrum-master coach — asks the questions that need answering before drafting a sprint file at sprints/YYYY-MM-DD.md. The draft is shown inline; the file isn't written until you sign off.

6. Weekly hygiene

/docs-update what's missing from the docs?
/docs-update any doc gaps from PRs merged this sprint?

docs-update analyzes recent project activity — team discussion, merged PRs in the configured repos, anything else AGENTS.md flags — and proposes documentation updates. It handles both high-level project docs (Confluence, a wiki, in-repo docs) and repo-associated technical docs that often drift behind the code. It's read-only until you confirm; the action it takes follows whatever update workflow you've described in AGENTS.md.

7. Reconciliation

When the local plans/ folder and external sources start to drift:

/plans sync plans — what's drifted, what should graduate, what's stale?

The plans skill's reconciliation mode compares local plans/ against Jira/Confluence and groups the findings by graduation candidates, drift, and stale items. Each acted-on item is its own confirmation; a single "apply everything" is too coarse for hygiene work that spans multiple sources.
