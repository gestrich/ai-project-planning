Getting started

A walkthrough for bootstrapping a fresh project onto the plugin and running the skills end-to-end.

1. Install the plugin

Install on whichever runtime you use — instructions live in the repo README:

  • Install on Claude Code
  • Install on OpenAI Codex

If you'd rather not install yet, you can point Claude Code at a local checkout:

claude --plugin-dir ~/path/to/ai-project-planning/plugin

2. Bootstrap an existing project

Open a Claude/Codex session inside the project you want to onboard, then say:

bootstrap this project for project planning

The bootstrap skill runs in three phases:

  1. Survey. Reads AGENTS.md, scans for existing notes/plans folders, peeks at commit history for hints of Jira/Confluence/Slack usage, and lists the project's GitHub remote. Read-only.
  2. Propose. Shows you a plan-of-changes — folders to create, files to move (via git mv), and the ## Project Planning section it wants to add to AGENTS.md. You answer a few clarifying questions inline.
  3. Apply. Only after you say "looks good" / "go ahead" / "apply", the skill makes the changes and summarizes what landed.

After bootstrap finishes, your project has a notes/ folder, a plans/ folder, a plans/sprints/ folder, and a populated ## Project Planning section in AGENTS.md. See the Configuration page for what that section looks like.

3. Capture a brain dump

Paste a transcript or brain dump into the conversation:

here's my brain dump — yeah so I was thinking about the auth migration, we need to figure out
whether to do the rollout in one shot or stage it, and there's that Jira ticket about the
session token storage that I haven't read yet...

The notes skill writes it verbatim to notes/YYYY-MM-DD-<slug>.md and confirms the path. No cleanup, no rewriting.

4. Ask what the plan is

what's the plan?

The plan skill reads your AGENTS.md, loads only the reference documents matching the sources you declared (Jira, Confluence, local plans/, Slack-as-context), and synthesizes a short, source-cited view of the project's direction.

5. Plan the week

what should I focus on this week?

The sprint skill gathers recent notes, reads the plan sources, looks at the previous sprint file (if any), and pulls a week of Slack activity from the configured channel. It then acts as a scrum-master coach — asks the questions that need answering before drafting a sprint file at plans/sprints/YYYY-MM-DD.md. The draft is shown inline; the file isn't written until you sign off.

6. Weekly hygiene

Two skills compose into a weekly cadence:

  • docs-update sweeps the configured Slack channel for the last week and proposes documentation updates against Confluence (or your local docs folder).
  • pr-review reviews an open PR against the project's documented conventions and surfaces both PR-side findings and doc-update suggestions in one pass.

Both skills are read-only by default. Anything they want to write — a Confluence page edit, a PR comment, a message to another team — waits for your explicit confirmation.

7. Reconciliation

When the local plans/ folder and external sources start to drift, ask:

sync plans — what's drifted, what should graduate, what's stale?

The plan skill's reconciliation mode compares local plans/ against Jira/Confluence and groups the findings by graduation candidates, drift, and stale items. Each acted-on item is its own confirmation; a single "apply everything" is too coarse for hygiene work that spans multiple sources.
